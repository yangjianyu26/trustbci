Data Contributor
Data Assetization Tools

Data Governance

Standardize data formats, unify metadata descriptions, ensure data integrity, enhance the data usability.

Data Pricing

Generate a recommended data price via analyzing factors such as data volume, signal quality, annotation density, and scenario uniqueness.

Data Desensitization

Detect and remove personally identifiable information such as identity IDs, device IDs and phone numbers from raw datasets.

Multi‑Dimensional Incentives

Financial Incentives

Contributors can receive direct monetary rewards based on data value and usage.

Reputation Incentives

Outstanding contributors are distinguished with official letters of appreciation, which can generate substantial public relations value and strengthen their credibility within the community.

Growth Incentives

Contributors may earn contribution points that help upgrade their membership levels consisting of Basic, Gold, Platinum, and Diamond. A higher level unlocks prioritized access to platform services.

Data Consumer
Usage Services

Controllable Data Synthesis

Platform generates a synthetic dataset as a surrogate that preserves the format and statistical characteristics of the original data, which enables consumers to expand their training samples and improve model performance.

Privacy-Preserving Data Query

Consumers submit data query statements the platform and only obtain sanitized outputs.

Federated Model Training

Platform coordinates multiple contributors to jointly train the model, with each contributor sharing only necessary updates rather than raw data.

Secure Data Room

Provide a privacy-preserving computing environment built on Trusted Execution Environment (TEE) technology that enables consumers to upload executable code for custom analysis without direct access to raw data.

Virtual Brain Model

Use uploaded research subject’ brain states along with relevant parameters to train a virtual brain model, and produce a digital twin that helps consumers better understand the subject’s brain.

General BCI Data Model

Provide a specialized large language model (LLM) tailored to the BCI domain, which is fine-tuned from a foundational LLM using the BCI data aggregated by the platform.

How to Use

  • First upload a dataset in Showcase 1 (or use the default name).
  • Supported: SELECT COUNT(*) FROM <table> WHERE <condition>
  • Condition can compare columns t1, t2, t3, t4, t5, t6, label with numbers or strings.
  • Example: SELECT COUNT(*) FROM bci_dataset_1 WHERE t1 > 8.5 AND label = 'Motor imagery'
  • Results include the noisy count and the original true count for demonstration.
Differential privacy budget ε=1.0
